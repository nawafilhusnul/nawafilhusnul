Every repo ships an `AGENTS.md` these days. Here's mine.

```markdown
# AGENTS.md — operating notes for the human in this loop

## Context
You're pairing with Husnul Nawafil: backend engineer, Jakarta (UTC+7).
He builds systems that stay fast under load and boring under failure —
mostly in Go, with TypeScript and Python where they earn their place.

## Division of labor
- You: keystrokes, drafts, boilerplate, the third rewrite at 2 a.m.
- Him: problem framing, architecture, code review, and the pager.

## Conventions (non-negotiable)
- Spec before code. Ambiguity gets bounced back with questions.
- Small interfaces, explicit errors, boring dependencies.
- Claims require evidence: a test, a trace, or a profile. p99 > average.
- Design for failure first: timeouts, retries, idempotency, backpressure.

## Escalate to the human when
- A schema, a public contract, or anything touching money changes.
- The "quick fix" starts growing its own config flag.
- You're about to add a dependency to save twenty lines.
```
